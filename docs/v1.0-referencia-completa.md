## 1. obtener_token

Valida credenciales y devuelve token de autorización.

**Parámetros:**

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `version` | Texto | `"1.0"` |
| `credenciales.id_organismo` | int | Código de organismo |
| `credenciales.id_usuario` | int | Código de usuario |
| `credenciales.password` | Texto | Password asignado |
| `credenciales.Hash` | Texto | Hash asignado |

**Respuesta:** `id_resp`, `respuesta`, `token`

| Código | Tipo | Descripción |
|--------|------|-------------|
| 01001 | OK | Token generado |
| 01002 | Error | Error al validar usuario o credenciales |
| 01003 | Error | Error interno al generar token |

> ⚠️ Diferencia con v2.x: no existe código 01004 (Versión inválida)

> ⚠️ Un token generado vía API solo puede consumir un método de API, NO sirve para E-Checkout.

## 2. solicitud_pago

Genera nueva operación. Solo para medios que NO sean tarjeta crédito/débito.

**Parámetros principales:**

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `version` | Texto | `"1.0"` |
| `tipo_operacion` | Set | Valor fijo `"op_pago"` |
| `credenciales` | Estructura | `id_organismo`, `token` |
| `operacion` | Estructura | Ver abajo |
| `fp` | Array | Formas de pago |
| `convenio` | int | Número de convenio |

**operacion (DatosOperacionPago):**

| Campo | Tipo | Obligatorio | Descripción |
|-------|------|:-----------:|-------------|
| `numero_operacion` | Texto | Opcional | Código externo del cliente |
| `identificador_externo_2` | Texto | Opcional | Identificador adicional |
| `identificador_externo_3` | Texto | Opcional | Identificador adicional |
| `id_moneda_operacion` | int | Sí | Valor fijo `1` |
| `monto_operacion` | decimal | Sí | Importe con 2 decimales |
| `opc_pdf` | bool | Opcional | Incluir PDF en respuesta (default: true) |
| `opc_fecha_vencimiento` | Fecha | Opcional | Fecha vencimiento boleta (AAAA-MM-DD) |
| `opc_devolver_qr` | bool | Opcional | Devolver imagen QR (default: false) |
| `opc_devolver_codbarras` | bool | Opcional | Devolver imagen código barras (default: false) |
| `detalle_operacion` | Array | Opcional | Ítems del importe |
| `pagador` | Estructura | Sí | Datos del pagador |
| `fecha_2do_venc` | Fecha | Opcional | Segundo vencimiento (AAAA-MM-DD) |
| `monto_operacion_2do_venc` | decimal | Opcional | Importe segundo vencimiento |

**pagador:** `nombre_pagador`, `apellido_pagador`, `fechanac_pagador`, `email_pagador`, `identificacion_pagador` (tipo_doc, numero_doc, cuit), `domicilio_pagador` (calle, numero, adicional, cp, ciudad, provincia, país=13), `telefono_pagador` (codigo, numero), `cbu_pagador`

**fp (FormaPago):** `id_fp`, `monto_fp`, `tarjeta` (numero, banco, vencimiento, codigo_seg, cuotas, titular, identificacion, fechanac, direccion)

**Respuesta:** `id_resp`, `respuesta`, `token`, `id_transaccion`, `id_organismo`, `convenio`, `numero_operacion`, `fp[]` → `codigo_pago_fp`, `codigo_barras_fp`, `fechavenc_fp`, `importe_fp`, `pdf` (base64), `codigo_barras_imagen` (base64 PNG), `qr_imagen` (base64 PNG), `respuesta_entidad_cobro`, `codigo_pmc`, `codigo_link`, `url_qr`, `cuit_cuenta`, `cbu_cuenta`

| Código | Tipo | Descripción |
|--------|------|-------------|
| 02001 | OK | Pago acreditado |
| 02002 | OK | Pago pendiente |
| 02003 | Error | Error al validar token |
| 02004 | Error | Pago cancelado/rechazado |
| 02005 | Error | Error interno |
| 02006 | Error | Error validar parámetro: [_parametro_] |
| 02007 | Error | Usuario canceló pago |
| 02008 | Error | No coinciden montos y detalles |
| 02009 | Error | Forma de pago no disponible |

> Nota: floats sin separador de miles, punto como decimal. Ej: `1234.12`

## 3. obtener_entidades_pago

Devuelve medios de pago disponibles para el Organismo.

**Parámetros:**

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `version` | Texto | `"1.0"` |
| `credenciales` | Estructura | `id_organismo`, `token` |
| `fp` | Array | Opcional. Filtro por IDs de forma de pago |

**Respuesta:** `id_resp`, `respuesta`, `token`, `id_organismo`, `fp[]`:

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `id_fp` | int | Código forma de pago |
| `nombre_fp` | Texto | Descripción |
| `tipo_fp` | Set | `tipo_fp_presencial`, `tipo_fp_credito`, `tipo_fp_debito`, `tipo_fp_prepago`, `tipo_fp_publicacion`, `tipo_fp_billetera`, `tipo_fp_transferencia` |
| `estado_fp` | Set | `estado_fp_activo`, `estado_fp_desactivado` |
| `logos_fp` | Estructura | `seguro_logos_fp` (HTTPS), `nombre_logos_fp` (HTTP) |
| `config_fp` | Estructura | `patron_config_fp`, `longitud_config_fp`, `validacion_config_fp`, `codseg_config_fp`, `codseg_long_config_fp`, `codseg_ubic_config_fp` |
| `adicional_fp` | Texto | Info adicional a solicitar (opcional) |
| `monto_minimo_fp` | decimal | Mínimo (opcional) |
| `monto_maximo_fp` | decimal | Máximo (opcional) |
| `tiempo_acreditacion_fp` | int | Horas de acreditación |

| Código | Tipo | Descripción |
|--------|------|-------------|
| 03001 | OK | Entidades devueltas |
| 03002 | Error | Error al validar token |
| 03003 | Error | Error interno |
| 03004 | Error | Error validar parámetro |

## 4. obtener_pagos

Lista operaciones con detalles y estado actual.

**Parámetros:**

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `version` | Texto | `"1.0"` |
| `credenciales` | Estructura | `id_organismo`, `token` |
| `pago` | Array | Criterios de búsqueda |

**Criterios de búsqueda:**

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `CodigoUnicoTransaccion` | int | ID único de operación |
| `ExternoId` | Texto | ID proporcionado por Organismo |
| `ExternoId_2` | Texto (100) | ID adicional |
| `ExternoId_3` | Texto (512) | ID adicional / protocolo interno |
| `Estado` | Set | A, O, V, C, R, P, D |
| `CodBarras` | Texto | Código de barras completo |
| `FechaPagoDesde` | Fecha | Inicio rango fecha pago |
| `FechaPagoHasta` | Fecha | Fin rango fecha pago |
| `FechaAcreditacionDesde` | Fecha | Inicio rango acreditación medio |
| `FechaAcreditacionHasta` | Fecha | Fin rango acreditación medio |
| `FechaNovedadAcreditacionDesde` | Fecha | Inicio rango acreditación ePagos |
| `FechaNovedadAcreditacionHasta` | Fecha | Fin rango acreditación ePagos |

**Estados:**
- `A` — Acreditada
- `O` — Adeudada
- `V` — Vencida
- `C` — Cancelada por usuario
- `R` — Rechazada por medio de pago
- `P` — Pendiente (sin selección de medio)
- `D` — Devuelto (importe reintegrado)

**Respuesta:** `id_resp`, `respuesta`, `token`, `id_organismo`, `pagos[]` con:
- `CodigoUnicoTransaccion`, `Externa`, `Externa_2`, `Externa_3`, `Convenio`, `Importe`, `Estado`
- `FechaPago`, `FechaAcreditacion`, `FechaNovedadAcreditacion`
- `FormaPago[]` → `Identificador`, `Tipo`, `Importe`, `CodigoPago`, `CodigoBarras`, `FechaVencimiento`
- `DatosPagador` → `Nombre`, `Apellido`, `Email`, `Identificacion`
- `DatosContribuyente` → `Nombre`, `Apellido`, `Email`, `Dominios`, `Identificacion`, `Turno[]`
- `PagosAdicionales[]` → `CodigoUnicoTransaccion`, `FormaPago`, `Monto`, `FechaPago`, `FechaNovedad`, `IdPago`
- `Recibo` (URL), `Url_QR`

> 💡 Para conciliación diaria usar `FechaNovedadAcreditacionDesde/Hasta`
> ⚠️ `DatosContribuyente` reemplaza a `DatosPagador` en algunos organismos

| Código | Tipo | Descripción |
|--------|------|-------------|
| 04001 | OK | Pagos devueltos |
| 04002 | Error | Error al validar token |
| 04003 | Error | Error interno |
| 04004 | Error | Rango de fechas supera límite |
| 04005 | Error | Error validar parámetro |
