# API v1.0 — Referencia de solicitud_pago

> Este documento detalla el método `solicitud_pago` tal como está documentado en la **v1.0** de la API.
> Para diferencias con v2.x, ver la sección final.

## Endpoint

| Entorno | URL WSDL |
|---------|----------|
| Sandbox | `https://sandbox.epagos.com/wsdl/index.php?wsdl` |
| Producción | `https://api.epagos.com/wsdl/index.php?wsdl` |

## Parámetros recibidos

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `version` | Texto | Valor fijo `"1.0"` |
| `tipo_operacion` | Set | Valor fijo `"op_pago"` |
| `credenciales` | Estructura DatosCredenciales | Datos de credenciales |
| `operacion` | Estructura DatosOperacionPago | Datos de la operación |
| `fp` | Array de FormaPago | Formas de pago |
| `convenio` | Numérico entero | Número de convenio asignado por ePagos |

### Estructura: DatosCredenciales

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `id_organismo` | Numérico entero | Código de organismo |
| `token` | Texto | El token |

### Estructura: DatosOperacionPago

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `numero_operacion` | Texto | Opcional. Código externo del cliente |
| `identificador_externo_2` | Texto | Opcional. Identificador adicional |
| `identificador_externo_3` | Texto | Opcional. Identificador adicional |
| `id_moneda_operacion` | Numérico entero | Valor fijo `1` |
| `monto_operacion` | Numérico decimal | Importe con dos decimales |
| `opc_pdf` | Booleano | Opcional. Incluir PDF en respuesta (default: true) |
| `opc_fecha_vencimiento` | Fecha (AAAA-MM-DD) | Opcional. Fecha vencimiento boleta |
| `opc_devolver_qr` | Booleano | Opcional. Devolver imagen QR (default: false) |
| `opc_devolver_codbarras` | Booleano | Opcional. Devolver imagen código de barras (default: false) |
| `detalle_operacion` | Array de DetalleOperacion | Detalles/ítems del importe |
| `pagador` | Array de Pagador | Datos del pagador |
| `fecha_2do_venc` | Fecha (AAAA-MM-DD) | Opcional. Fecha segundo vencimiento |
| `monto_operacion_2do_venc` | Numérico decimal | Opcional. Importe segundo vencimiento |

### Estructura: DetalleOperacion

| Nombre | Tipo | Descripción |
|--------|------|-------------|

## Respuesta

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `id_resp` | Numérico entero | Código de respuesta |
| `respuesta` | Texto | Descripción de la respuesta |
| `token` | Texto | Token único de la solicitud |
| `id_transaccion` | Numérico entero | Código único de operación ePagos |
| `id_organismo` | Numérico entero | Número de organismo |
| `convenio` | Numérico entero | Número de convenio |
| `numero_operacion` | Numérico entero | Número interno de operación |
| `fp` | Array de RespuestaFormaPago | Respuesta por forma de pago |

### Estructura: RespuestaFormaPago

| Nombre | Tipo | Descripción |
|--------|------|-------------|
| `codigo_pago_fp` | Numérico entero | Código corto para pago efectivo/homebanking |
| `codigo_barras_fp` | Texto | Código de barras completo |
| `fechavenc_fp` | Fecha | Fecha vencimiento (AAAA-MM-DD) |
| `importe_fp` | Numérico decimal | Importe de la operación |
| `pdf` | Base64 | PDF del comprobante |
| `codigo_barras_imagen` | Base64 | Imagen código de barras (PNG, si `opc_devolver_codbarras = true`) |
| `qr_imagen` | Base64 | Imagen QR (PNG, si `opc_devolver_qr = true`) |
| `respuesta_entidad_cobro` | Texto | Detalle respuesta del medio de pago |
| `codigo_pmc` | Texto | Código publicación Pago Mis Cuentas |
| `codigo_link` | Texto | Código publicación Red Link |
| `url_qr` | Texto | URL para pago en línea con tarjeta |
| `cuit_cuenta` | Numérico entero | CUIT de la cuenta |
| `cbu_cuenta` | Texto | CBU/CVU o alias de cuenta destino DEBIN |

## Respuestas posibles

| id_resp | Tipo | Respuesta |
|---------|------|-----------|
| 02001 | Correcta | Pago acreditado |
| 02002 | Correcta | Pago pendiente |
| 02003 | Error | Error al validar el token |
| 02004 | Error | Pago cancelado / rechazado |
| 02005 | Error | Error interno al intentar procesar el pago |
| 02006 | Error | Error al validar el parámetro: [_parametro_] |
| 02007 | Error | El usuario canceló el pago |
| 02008 | Error | No coinciden los montos y los detalles |
| 02009 | Error | La forma de pago [_parametro_] no se encuentra disponible |

## Nota sobre tipos numéricos

Los campos de tipo float se envían sin separadores de miles y con punto (`.`) como separador decimal. Ejemplo: `1234.12`. Si es entero, puede enviarse sin `.00`.

## Diferencias clave v1.0 vs v2.x

| Aspecto | v1.0 | v2.x (v2.0 / v2.1 / v2.4 / v2.5) |
|---------|------|----------------------------------|
| WSDL path | `/wsdl/index.php?wsdl` | `/wsdl/2.x/index.php?wsdl` |
| Valor `version` | `"1.0"` | `"2.0"` |
| Campo `tipo_operacion` | Presente (`"op_pago"`) | Verificar en doc v2.x |
| Estructura general | Parámetros top-level separados | Similar, posibles campos nuevos |
| Métodos nuevos (QR, débito) | ❌ No disponibles | ✅ Desde v2.1 |
